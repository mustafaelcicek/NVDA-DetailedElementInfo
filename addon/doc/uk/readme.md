# Детальна інформація про елемент

Потужний додаток NVDA, який надає детальну інформацію про доступність елементів HTML на веб-сторінках.

## Огляд

«Детальна інформація про елемент» – це додаток для NVDA, який дозволяє перевіряти елементи HTML та переглядати їхні атрибути доступності, ролі ARIA та семантичну інформацію. Це особливо корисно для веб-розробників, аудиторів доступності та користувачів NVDA, які хочуть зрозуміти, як програми читання з екрана сприймають веб-контент.

## Функції

- **Показ ролей ARIA**: перегляд обчисленої ролі ARIA для будь-якого HTML-елемента
- **Показ доступних назв**: перегляд доступної назви та її джерела (aria-label, альтернативний текст, вміст тощо)
- **Виявлення семантичних невідповідностей**: виявлення випадків, коли явно задані ролі ARIA суперечать вбудованій семантиці HTML
- **Переглянути атрибути**: переглянути всі атрибути IA2 та властивості ARIA
- **Показати шлях DOM**: переглянути стислий шлях DOM для елемента, на якому встановлено фокус
- **Навігація по батьківських елементах**: переходити між батьківськими елементами, щоб зрозуміти структуру документа
- **Відстеження стану**: перегляд станів елементів (розгорнуто, позначено, вибрано тощо) як атрибутів ARIA
- **Виявлення джерела**: визначення походження доступних імен (aria-label, contents, alt, title тощо)
- **Достовірність назви джерела**: Перевірте, наскільки додаток впевнений щодо походження назви зі спеціальними можливостями (певне, ймовірне, невідоме)
- **Перегляд різних ролей**: Порівняйте явно вказану роль ARIA, вбудовану роль HTML, роль NVDA та остаточну виведену роль у вигляді паралельного порівняння
- **Роль MSAA та заголовок рівня**: Перегляд необробленої ролі MSAA та, для заголовків, заголовка рівня
- **Атрибути похідного стану**: Перегляд атрибутів, подібних до ARIA, похідних від станів NVDA (розгорнутий, позначений, вибраний тощо)
- **Настроювані поля**: у налаштуваннях додатка виберіть, які саме поля мають показуватись у звіті, скориставшись опціями «Вибрати все» / «Скинути все»
- **Розширена підтримка — Chrome Bridge (експериментальна версія)**: Підключення до супутнього розширення Chrome для безпосереднього отримання реальних даних DOM сторінки, атрибутів HTML та активних властивостей
- **Аналіз за допомогою Gemini (експериментально)**: запитайте Google Gemini просте пояснення того, що являє собою сфокусований елемент і як його можна використовувати.

## Гарячі клавіші

| Комбінація | Дія |
|----------|--------|
| NVDA+Shift+F1 | Показати деталі елемента у фокусі |

> Примітка: Цю комбінацію клавіш можна змінити в NVDA меню **Параметри > Жести вводу**.

## Як використовувати

1. Перейдіть до будь-якого HTML-елемента на веб-сторінці за допомогою клавіші Tab або клавіш зі стрілками
2. Натисніть NVDA+Shift+F1, щоб показати детальну інформацію про елемент, на якому встановлено фокус
3. Відкриється діалог повідомлення, в якому можна переглядати такі дані:
   - Назва тегу елемента та кількість атрибутів
   - Доступна назва та її джерело
   - Обчислена роль ARIA
   - Джерело ролі: вбудоване чи явне
   - Семантичні невідповідності
   - Усі необроблені атрибути IA2
   - Шлях DOM елемента, на якому встановлено фокус
   - Батьківські елементи вище в дереві DOM


## Інформація, що показується

### Доступна назва
Назва, яку екранні зчитувачі озвучують для цього елемента.

### Назва Джерела
Звідки походить доступна назва:
- **aria-label**: Явна мітка ARIA
- **aria-labelledby**: Посилання на інший елемент
- **вміст**: Текстовий вміст елемента
- **alt**: Альтернативний текст (для зображень)
- **value**: атрибут «value» елемента
- **title**: Атрибут заголовка
- **placeholder**: Текст заповнювача

### Обчислювана роль
Роль ARIA визначається з:
1. Явно вказані ролі XML (атрибут role)
2. Вбудована семантика HTML
3. Прив’язка типів елементів керування NVDA
4. Невідомо (резервний варіант)

### Семантичне джерело
Чи походить ця роль від:
- **вбудована семантика HTML**: неявна семантика HTML (наприклад, <button> → роль «button»)
- **явна роль**: перевизначення атрибута role
- **надлишкова явна роль**: значення атрибута role збігається з вбудованою семантикою
- **невідомо**: неможливо визначити

### Внутрішня семантична невідповідність
Вказує, чи явна роль ARIA конфліктує з власною HTML-семантикою елемента.

### Шлях DOM
Короткий технічний шлях, що допомагає визначити розташування елемента в структурі сторінки (наприклад: `main > form#login > input[name=«email»]`).

## Налаштування

Відкрийте **меню NVDA > Параметри > Налаштування > Детальна інформація про елемент**, щоб налаштувати додаток:

— **Виберіть поля для показу**: вмикайте або вимикайте окремі поля звіту — зведення тегів, доступна назва, джерело назви та достовірність, тег і шлях DOM, явні / власні / NVDA / підсумкові ролі, семантичне джерело та невідповідність, необроблені атрибути IA2, атрибути, похідні від станів NVDA, роль MSAA та батьківські елементи.
- **Вибрати все / Очистити все**: Увімкнути або вимкнути всі поля одночасно.
- **Увімкнути розширену підтримку (розширення ШІ та Chrome)**: Вмикає експериментальні функції Chrome Bridge та ШІ, описані нижче.
- **Ключ API Gemini**: Введіть власний ключ API Google Gemini, щоб можна було запустити аналіз за допомогою ШІ.

## Розширена підтримка — Chrome Bridge (експериментальна версія)

Додаток «Детальна інформація про елемент» може за бажанням підключатися до супутнього розширення Chrome, щоб зазирнути за межі віртуального буфера NVDA та зчитати реальний DOM сторінки.

- Коли увімкнено «Розширену підтримку», додаток запускає полегшений локальний сервер (`127.0.0.1:63333`), який взаємодіє з розширенням Chrome за допомогою протоколу HTTP з тривалим опитуванням.
- Після цього у звіті з’являється розділ **«Атрибути DOM активного елемента»**, що містить атрибути та властивості HTML у режимі реального часу, отримані з Chrome.
- Для цього необхідно встановити відповідне розширення Chrome. Ця функція є експериментальною та активно розробляється.

## Аналіз ШІ за допомогою Gemini (експериментально)

Якщо увімкнено розширену підтримку та налаштовано ключ API Gemini, у звіті доступна дія **Запитати коментар ШІ**.

- Зібрані дані DOM надсилаються до Google Gemini, який повертає коротке пояснення простою мовою про те, що це за елемент, яке його призначення та як користувач екранного зчитувача може взаємодіяти з ним.
- Для цього потрібен ваш власний ключ API Google Gemini, який потрібно ввести в налаштуваннях. Ця функція є експериментальною.

## Технічні деталі

Цей додаток використовує **нативну архітектуру без залежностей**:
- Стандартні бібліотеки Python (`http.server`, `urllib.request`) для уникнення проблем із залежностями `pip`.
- Інтерфейси IAccessible2 (IA2) для атрибутів доступності
- Об'єктну модель та типи елементів керування NVDA
- Специфікації WAI-ARIA 1.2
- Ролі Windows MSAA
- **Тривале опитування HTTP**: полегшений спосіб зв’язку із сервіс-воркером розширення Chrome (Manifest V3), який підтримується в активному стані за допомогою періодичних сигналів від контент-скриптів.
